# Render + Supabase Migration

Audit Angel runs on Render as a Node web service and uses Supabase Postgres for persistent data.

## Required Render Environment Variables

- `NODE_ENV=production`
- `NODE_VERSION=22.22.0`
- `DATABASE_URL`: Supabase Postgres connection string.
- `SESSION_SECRET`: generated by Render unless rotating intentionally.
- `DEFAULT_USER_EMAIL`: fallback seeded login if the restored database has no matching user.
- `DEFAULT_USER_PASSWORD`: fallback seeded password.

## Move Existing Railway Data

From a machine that can reach both databases:

```bash
pg_dump "$RAILWAY_DATABASE_URL" --no-owner --no-acl > backups/audit-angel-railway-$(date +%F-%H%M).sql
psql "$SUPABASE_DATABASE_URL" < backups/audit-angel-railway-YYYY-MM-DD-HHMM.sql
```

Verify the restore includes these tables:

```sql
SELECT table_name
FROM information_schema.tables
WHERE table_schema = 'public'
ORDER BY table_name;
```

Expected application tables:

- `users`
- `investigations`
- `investigation_members`
- `pins`
- `pin_images`
- `dismissed_anomalies`
- `bonus_configs`
- `user_preferences`
- `session`

Confirm pinned image data survived:

```sql
SELECT COUNT(*) AS image_count,
       COUNT(*) FILTER (WHERE data_url IS NOT NULL AND data_url <> '') AS images_with_data
FROM pin_images;
```

## Deploy on Render

1. Connect `ProgrammingDrak/audit-angel`.
2. Use the root `render.yaml` blueprint.
3. Set all `sync: false` values in the Render dashboard.
4. Deploy and verify `/api/health`.
5. Log in, open existing investigations, verify pins/images/preferences, create and delete a test pin, then export JSON.

Keep the Railway database and SQL dump until Render and Supabase smoke tests pass.
